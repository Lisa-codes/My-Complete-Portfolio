Padel Tennis Mobile App

The Padel Tennis mobile app was designed to enhance the experience of playing padel tennis at the Village Market. This app allows users to effortlessly book their sessions and make payments, integrating a seamless user interface with a QR code system to easily add friends and enjoy games together. The app also simplifies the booking process, making it easier for users to explore match types and tournaments, while tracking their scores for each game played.

Project Duration: May 2024 - June 2024 (1 month)

Role: Lead UI/UX Designer

Design Process


User Personas

In order to empathize with the user during the design process, I crafted personas based on real life users who play the game to better craft the user experience while highlighting key user motivations and goals.


User Flow Diagram

In the Padel Tennis mobile app, the user flow begins with the Sign-Up process, where users are prompted to create an account by entering their basic information, such as name, email, and password, or by signing in using third-party authentication options like Google or Email. Once signed up, users proceed to the Home Screen, which presents available tennis courts, match schedules, and upcoming tournaments. From here, they can navigate to the Booking Section, where they select a court, choose a date and time, and confirm their booking.

Next, users are directed to the Payment Screen, where they can view the booking summary and make payments through integrated payment methods such as credit/debit cards, mobile wallets, or other available options. Upon successful payment, the user receives an Instant Notification confirming their booking details, which includes the court number, time slot, and any other relevant instructions. The app also allows users to receive notifications for upcoming matches or changes to their booking. This seamless flow ensures a user-friendly experience, from account creation to court booking and payment completion, with real-time notifications keeping the user informed throughout.

Initial Wireframes

During the ideation stage, I crafted paper wireframes as a foundation for the design process. The paper wireframes helped with understanding how to layout the components when making digital wireframes and empathize with the user and how they would go through the user journey.

Gathering insights

After the first design stage, I conducted user interviews to gather any pain points from the user and further iterate the designs. From these interviews, I gathered data that I put in an affinity diagram to break down insights gained from users.

  • 25% of the users found the booking process to be confusing and time-consuming.
  • 50% of users indicated that there was a need for a uniformity in CTA buttons to avoid user confusion.
  • 25% of the users requested more sign up/sign in options such as use of Google auth and email was requested.

High-Fidelity Mockups

Based on the insights gathered designs were iterated addressing the pain points with the following solutions.

  • Using an internal design system to address uniformity of components.
  • Streamlining the booking process using time slot filters to show the particular courts available during particular times.
  • Adding sign up/sign in options by use of Google auth and email as was requested .

Challenges & Solutions

One of the primary challenges of this project was the need to deliver a fully functional app within a tight timeline. To address this, we prioritized the development of core features such as the booking system, QR code integration, and score tracking. Rapid prototyping allowed for quick iterations, ensuring that each design element was validated before moving on to the next stage.

The final product successfully streamlined the booking process, enhanced user interaction with QR code functionalities, and provided an intuitive interface for tracking game scores.

Future Improvements

As the Lead UI/UX Designer, I am always looking for ways to enhance the user experience. Moving forward, I plan to improve the score tracking feature by allowing users to add scores directly through the game history page. This will provide more accurate tracking for individual games, ensuring that users can keep detailed records of their performance over time.
